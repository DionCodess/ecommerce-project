# CS491 E-commerce Web App

This is a TypeScript React e-commerce project for CS491 Software Engineering. It includes a product catalog, cart, dummy checkout, admin dashboard, Firebase deployment config, Firestore rules/indexes, tests, and documentation.

## Tech Stack

- React + TypeScript with strict compiler settings
- Vite for local development and production builds
- Object-oriented domain, service, and repository layers
- Firebase Hosting for deployment
- Firestore for product and order persistence
- Firebase Auth for admin identity support
- Vitest for focused unit tests

## Architecture

Business rules live outside React components:

- `src/domain`: typed domain classes such as `Product`, `Cart`, `CartItem`, `Order`, `Money`, and `UserProfile`.
- `src/services`: OOP service classes such as `ProductService`, `CartService`, `CheckoutService`, and `AdminService`.
- `src/repositories`: repository interfaces plus local and Firestore implementations.
- `src/pages` and `src/components`: UI rendering and user interactions.
- `src/config`: Firebase client setup.

React hooks are used for UI state and lifecycle because React requires them. Cart totals, checkout creation, product validation, order status changes, and persistence are handled by classes.

See `DESIGN.md` for UML diagrams, data model diagrams, sequence diagrams, and design rationale.

## User Stories

### Shopper Stories

- As a shopper, I want to browse active products so that I can see what is available for purchase.
Acceptance criteria: products show name, description, category, image, price, and inventory count.
- As a shopper, I want to search products by text so that I can quickly find relevant items.
Acceptance criteria: search matches product name, description, or category.
- As a shopper, I want to filter products by category so that I can narrow the catalog.
Acceptance criteria: the storefront supports all categories and an all-categories option.
- As a shopper, I want to open a product detail page so that I can review an item before adding it to my cart.
Acceptance criteria: the detail page shows product description, price, inventory, and an add-to-cart action.
- As a shopper, I want to add products to my cart so that I can prepare an order.
Acceptance criteria: adding an existing cart item increases quantity and updates the cart count.
- As a shopper, I want to update cart quantities and remove items so that my cart reflects what I intend to buy.
Acceptance criteria: totals update immediately after quantity changes or removals.
- As a shopper, I want to complete a dummy checkout so that I can receive an order confirmation without using real payment information.
Acceptance criteria: checkout requires customer details, creates a paid order, clears the cart, and displays an order id.
- As a shopper, I want to see an AI-style product recommendation so that I can discover a useful item faster.
Acceptance criteria: the recommendation uses currently available products and works without a paid AI API.

### Admin Stories

- As an admin, I want to sign in before managing the store so that catalog and order operations are protected.
Acceptance criteria: Firebase mode uses Firebase Auth and Firestore rules check the admin role.
- As an admin, I want to view dashboard metrics so that I can understand product and order activity.
Acceptance criteria: the dashboard shows product count, active product count, order count, and revenue.
- As an admin, I want to create and update products so that the storefront catalog stays current.
Acceptance criteria: product input includes id, name, description, category, image URL, price, inventory, active status, and featured status.
- As an admin, I want to delete products so that unavailable items can be removed from the catalog.
Acceptance criteria: deleted products no longer appear in the product list.
- As an admin, I want to view orders so that I can track dummy checkout activity.
Acceptance criteria: orders show customer name, order id, total, date, and status.
- As an admin, I want to update order status so that fulfillment progress is visible.
Acceptance criteria: order status can be changed among pending, paid, processing, shipped, and cancelled.

### System And Deployment Stories

- As a developer, I want strict TypeScript checks so that untyped data flow is caught before deployment.
Acceptance criteria: `npm run typecheck` passes with strict settings and no implicit `any`.
- As a developer, I want deployment controlled by repository config so that the live app can be reproduced.
Acceptance criteria: Firebase Hosting, Firestore rules, Firestore indexes, and GitHub Actions workflow are committed.
- As a developer, I want seed data to be generated by code so that the demo catalog does not require manual database editing.
Acceptance criteria: `scripts/seedFirestore.ts` writes typed sample products to Firestore.

## Local Setup

Install Node.js 22 or newer, then run:

```bash
npm install
cp .env.example .env
npm run dev
```

For local-only demo mode, set this in `.env`:

```bash
VITE_USE_FIREBASE=false
```

For Firebase-backed mode, keep `VITE_USE_FIREBASE=true` and fill in the Firebase web app values in `.env`.

## Quality Commands

```bash
npm run typecheck
npm run lint
npm run test
npm run build
```

The project is configured to reject implicit `any` and explicit `any`.

## Firebase Setup

One-time manual setup that cannot be represented safely in committed code:

1. Create a Firebase project.
2. Enable Firebase Hosting.
3. Enable Firestore.
4. Enable Firebase Auth.
5. Create an admin user in Firebase Auth.
6. Create a Firestore document at `users/{adminUserUid}` with `{ "role": "admin" }`.
7. Register a web app and copy its config into `.env`.
8. Replace `replace-with-firebase-project-id` in `.firebaserc`.
9. Add GitHub repository secrets listed below.

Everything else is controlled by repo files:

- `firebase.json`
- `.firebaserc`
- `firestore.rules`
- `firestore.indexes.json`
- `.github/workflows/deploy.yml`
- `.env.example`
- `scripts/seedFirestore.ts`

## Required GitHub Secrets

The deployment workflow expects:

```bash
FIREBASE_PROJECT_ID
FIREBASE_TOKEN
VITE_FIREBASE_API_KEY
VITE_FIREBASE_AUTH_DOMAIN
VITE_FIREBASE_PROJECT_ID
VITE_FIREBASE_STORAGE_BUCKET
VITE_FIREBASE_MESSAGING_SENDER_ID
VITE_FIREBASE_APP_ID
```

Generate `FIREBASE_TOKEN` with:

```bash
npx firebase login:ci
```

## Deployment

The deploy workflow runs on pushes to `main` and performs:

```bash
npm ci
npm run typecheck
npm run lint
npm run test
npm run build
npx firebase-tools deploy --only hosting,firestore:rules,firestore:indexes
```

Manual deploy from a configured local machine:

```bash
npm run deploy
```

## Seeding Demo Data

To seed Firestore with the sample catalog, provide a service account JSON as an environment variable:

```bash
export FIREBASE_SERVICE_ACCOUNT_JSON='{"project_id":"...","client_email":"...","private_key":"..."}'
npm run seed:firestore
```

The seed script uses typed `Product` objects from `src/data/sampleProducts.ts`.
